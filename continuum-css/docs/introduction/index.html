Introduction

What is Continuum?

Continuum is a powerful yet lightweight JavaScript framework built around the premise that all CSS features should always be available to you from within your markup.

Tell Me Moreâ€‹

Let's start with an example.

Traditionally, whenever a developer needed to style anything on the web, they had to first write some CSS:

CSS
#body {
    background-color: #5d977a;
    color: white;
}
#main {
    position: absolute;
    left: 50%;
    top: 50%;
    transform: translate(-50%, -50%);
}
.title {
    font-size: 30px;
    font-weight: 750;
    text-decoration-line: underline;
}

Then, choose between using a <link> or <style> tag to make these styles available:

<link
    ...
    href="styles.css"
/>
<style type="text/css">
    .body { ...
</style>

And finally, apply the appropriate tags, classes, and IDs to style the page's content as envisioned:

HTML
<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8" />
        <meta
            name="viewport"
            content="width=device-width, initial-scale=1.0"
        />
        <link
            rel="stylesheet"
            type="text/css"
            href="styles.css"
        />
    </head>
    <body id="body">
        <div id="main">
            <h1 class="title">Hello World!</h1>
        </div>
    </body>
</html>

All with the end goal of seeing...

This approach, though still viable, does not lend itself to:

  • Accelerating development.
  • Reducing decision fatigue.
  • And most importantly, retaining creative control as a designer.

This is where Continuum steps in.

It's drop-in web development solution that can be easily integrated into any existing web application with minimal effort, designed to always enhance functionality without requiring any restructuring of existing codebases.

For example, to still get the above page, only this time whilst leveraging Continuum:

HTML
<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8" />
        <meta
            name="viewport"
            content="width=device-width, initial-scale=1.0"
        />
        <script
            defer
            src="https://cdn.jsdelivr.net/npm/continuum-css@latest/dist/cdn.min.js"
        ></script>
    </head>
    <body class="b622=#5d977a c320=w097">
        <div class="p963=a770 l052=50% t792=50% t526=t310(-50%;-50%)">
            <h1 class="f582=30px f673=750 t138=u441">Hello World!</h1>
        </div>
    </body>
</html>

Same result, but with the added benefit of never having to write a single line of custom CSS, while still being able to create your own custom styles.

Plus, at least 30% less typing is always welcome!

â€œThat looks like a messâ€ is probably what you're thinking. And if you are, you're not wrong. It is somewhat unconventional. But once you try it, and after a while actually start to see beyond the matrix, youâ€™ll notice:

  • Accelerated Development

Significantly enhance the efficiency of styling HTML by using classes that encapsulate common styling properties directly in the markup itself. This approach not only saves time as well as reduce the need for writing custom CSS, it also ensures consistency across the application. Moreover, the available community support and extensive up-to-date documentation facilitate quicker problem-solving and learning.

  • Improved Maintainability

This method not only speeds up the development process but also promotes consistency across the application. Additionally, it encourages a more modular approach to design, making it easier to maintain and update styles. By minimizing context switching between HTML and CSS files, developers can focus more on other time-consuming tasks, which ultimately leads to faster project completion while still maintaining high-quality standards.

  • Reduced Decision Fatigue

For example, instead of writing custom styles for margins, padding, and colors, a developer can simply use unreserved classes like m276=a793, p427=10vh, and c320=r054. This not only speeds up the development process but also reduces cognitive load, allowing developers to focus on app functionality and user experience rather than getting bogged down in deliberating over which styles to apply or what custom CSS to create for each component.

  • Minimised Complexity

Significantly minimise complexity in styling HTML by implementing a functional approach to design. Instead of managing multiple style-sheets and writing extensive custom CSS for each component, developers can both optimise their workflow and clean their codebase by simply applying classes directly in the markup. This approach allows for changes to be made quickly by only adjusting the classes themselves as opposed to diving into separate CSS files.

  • Retained Creative Control

The absence of arbitrary constraints requiring you to choose your styles from a predefined set of pre-approved design systems, empowers designers to make real-time adjustments, which fosters a collaborative environment where both developers and designers can contribute to the visual aspects without stepping on each other's toes. Ultimately, this unique class structure promotes retaining creative control by providing flexibility and efficiency in the design process.

The Big Questionâ€‹

And now for the big question you may have had for a while... â€œWhy not just use inline styles?â€

While it may seem that using Continuum classes is akin to inline styles â€” since both involve applying styles directly to elements â€” there are significant advantages to using the custom class approach:

  • Firstly, our classes facilitate responsive design. Unlike inline styles, which do not support media queries or any other such CSS features, our classes allow for the easy implementation of responsive utilities, enabling the creation of fully adaptable interfaces.

  • Secondly, our classes can effectively manage various states such as hover and focus. Inline styles lack the capability to target these or any other states, but Continuum classes provide state variants that simplify targetted styling for varying user interactions.

  • Lastly, our classes allow for the use of custom helpers, like the @cc-grab at-rule, which extend the capabilities of standard CSS. These tailored helpers enhance workflows and improve overall code quality by delivering specific design needs that cannot be met when using inline styles alone.

For example, the following component is entirely responsive and features a button that incorporates hover and focus specific styles, all constructed only using Continuum classes:

HTML
<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8" />
        <meta
            name="viewport"
            content="width=device-width, initial-scale=1.0"
        />
        <script
            defer
            src="https://cdn.jsdelivr.net/npm/continuum-css@latest/dist/cdn.min.js"
        ></script>
        <style type="text/continuum-css">
            #toggle:c319 ~ dialog {
                d497: b102;
            }
        </style>
    </head>
    <body
        class="f649=m091 f582=20px d497=f726 a058=c112 j649=c112 h609=100vh m276=0 p427=0 w968=100% b622=#E5E5F7 b657=r866(e688,f294,a141,10px,10px;#F7B267;#F7B267,50%;#000,50%) b595=10px,10px"
    >
        <input
            type="checkbox"
            aria-hidden="true"
            id="toggle"
            class="d497=n068"
        />
        <label
            for="toggle"
            class="b565=#F7B267 o825:h858=0.95 b565:f194=l900 c320=w097 p427=0.5em,1em b852=4px f673=900"
            >Open Dialog</label
        >
        <dialog
            class="a465=appear,350ms,e792,1 m592=500px b852=4px @k597(appear)0%[o825=0,t526=t548(-10px)]"
        >
            <p>
                Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do
                eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut

In summary, Continuum classes not only enhance consistency and improve responsiveness, but also streamline state-management and encourage developer creativity; making them the superior choice for modern web development.

tip

Continuum is designed to simplify development processes wherever it can.

One of the ways it accomplishes this is through the custom cc-grab attribute, which allows developers to template their classes without the help of some custom JavaScript.

For example, to get the same result as seen in the example above, but without the common <input> tags' classes being repeated:

<label
    ...
    class="c320=#fff ..."
    >Sign up</label
>
<div class="w968=60% ...">
    <input
        ...
        placeholder="Username"
        cc-grab
    />
    <input
        ...
        placeholder="Email"
        cc-grab
    />
    <input
        ...
        placeholder="Password"
        cc-grab
    />
</div>
<button class="w968=60% ...">Sign up</button>

Now at runtime, the inputs' parent element's classes will be grabbed and prepended to the inputs' own classes, before said parent is removed from the DOM.

Want to keep the parent element itself for whatever reason? Then use the corresponding custom @cc-grab at-rule. For example:

<label
    ...
    class="c320=#fff ... @cc-grab[w968=60%,...]"
    >Sign up</label
>
<input
    ...
    placeholder="Username"
    cc-grab
/>
<input
    ...
    placeholder="Email"
    cc-grab
/>
<input
    ...
    placeholder="Password"
    cc-grab
/>
<button class="w968=60% ...">Sign up</button>

This time, only classes that have been marked as grabbable via the use of @cc-grab will be grabbed (and also removed) from the inputs' parent element and prepended to the inputs' own classes, with said parent then still remaining in the DOM to function or style as intended.
